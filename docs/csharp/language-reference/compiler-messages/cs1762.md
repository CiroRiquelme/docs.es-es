# <a name="compiler-warning-level-1-cs1762"></a><span data-ttu-id="51da0-102">Advertencia del compilador (nivel 1) CS1762</span><span class="sxs-lookup"><span data-stu-id="51da0-102">Compiler Warning (level 1) CS1762</span></span>

<span data-ttu-id="51da0-103">Se ha creado una referencia al ensamblado de interoperabilidad "\<ensamblado1>" insertado debido a una referencia indirecta a dicho ensamblado desde el ensamblado "\<ensamblado2>".</span><span class="sxs-lookup"><span data-stu-id="51da0-103">A reference was created to embedded interop assembly '\<assembly1>' because of an indirect reference to that assembly from assembly '\<assembly2>'.</span></span> <span data-ttu-id="51da0-104">Considere cambiar la propiedad "Incrustar tipos de interoperabilidad" en uno de los ensamblados.</span><span class="sxs-lookup"><span data-stu-id="51da0-104">Consider changing the 'Embed Interop Types' property on either assembly.</span></span>  
  
 <span data-ttu-id="51da0-105">Ha agregado una referencia a un ensamblado (ensamblado1) cuya propiedad `Embed Interop Types` está establecida en `True`.</span><span class="sxs-lookup"><span data-stu-id="51da0-105">You have added a reference to an assembly (assembly1) that has the `Embed Interop Types` property set to `True`.</span></span> <span data-ttu-id="51da0-106">Esto indica al compilador que incruste la información de tipo de interoperabilidad desde este ensamblado.</span><span class="sxs-lookup"><span data-stu-id="51da0-106">This instructs the compiler to embed interop type information from that assembly.</span></span> <span data-ttu-id="51da0-107">Sin embargo, el compilador no puede incrustar información de tipo de interoperabilidad desde este ese ensamblado porque otro ensamblado al que ha hecho referencia (ensamblado2) también hace referencia a este ensamblado (ensamblado1) y tiene la propiedad `Embed Interop Types` establecida en `False`.</span><span class="sxs-lookup"><span data-stu-id="51da0-107">However, the compiler cannot embed interop type information from that assembly because another assembly that you have referenced (assembly2) also references that assembly (assembly1) and has the `Embed Interop Types` property set to `False`.</span></span>  
  
> [!NOTE]
> <span data-ttu-id="51da0-108">Para el compilador de línea de comandos, el establecimiento de la propiedad `Embed Interop Types` de una referencia a ensamblado en `True` equivale a hacer referencia al ensamblado utilizando la opción `/link`.</span><span class="sxs-lookup"><span data-stu-id="51da0-108">Setting the `Embed Interop Types` property on an assembly reference to `True` is equivalent to referencing the assembly by using the `/link` option for the command-line compiler.</span></span>

- <span data-ttu-id="51da0-110">Para incrustar información de tipo de interoperabilidad para ambos ensamblados, establezca la propiedad `Embed Interop Types` de todas las referencias a ensamblado1 en `True`.</span><span class="sxs-lookup"><span data-stu-id="51da0-110">To embed interop type information for both assemblies, set the `Embed Interop Types` property on all references to assembly1 to `True`.</span></span> <span data-ttu-id="51da0-111">Para obtener más información sobre cómo establecer esa propiedad, vea [Tutorial: Insertar los tipos de los ensamblados administrados](../../../standard/assembly/embed-types-visual-studio.md).</span><span class="sxs-lookup"><span data-stu-id="51da0-111">For more information about how to set that property, see [Walkthrough: Embedding Types from Managed Assemblies](../../../standard/assembly/embed-types-visual-studio.md).</span></span>
